## üé® VISUAL GUIDELINES

### Screen Recording Tips
1. **Clean Desktop**: Remove clutter, use dark mode
2. **Large Font**: Increase terminal/browser font size for readability
3. **Smooth Transitions**: Use fade transitions between sections
4. **Highlight Key Points**: Use arrows/circles to highlight important UI elements
5. **Show Results**: Let results display for 3-5 seconds before moving on

### Pacing
- **Act 1 (Problem)**: Fast-paced, urgent tone
- **Act 2 (Solution)**: Slower, explanatory, let demos breathe
- **Act 3 (Impact)**: Inspirational, building momentum
- **Act 4 (CTA)**: Confident, inviting

### Music Suggestions
- **Act 1**: Tense, problem-focused (low tempo)
- **Act 2**: Uplifting, discovery (medium tempo)
- **Act 3**: Inspirational, epic (building tempo)
- **Act 4**: Confident, modern (high energy)

## üìù SCRIPT NOTES

### Tone Guidelines
- **Confident but not arrogant**: "We're solving real problems"
- **Technical but accessible**: Explain concepts simply
- **Inspiring but realistic**: Show what's possible today
- **Collaborative not competitive**: "We complement the ecosystem"

### Storytelling Elements
1. **Hero's Journey**: Developer (hero) faces complexity (villain), finds CronoPay (weapon), transforms workflow (victory)
2. **Before/After**: Always show the contrast
3. **Show, Don't Tell**: Let demos speak for themselves
4. **Emotional Connection**: Developers FEEL the frustration, then the relief

### Delivery Tips
- **Pause after key points**: Let them sink in
- **Vary pace**: Slow for complex concepts, fast for excitement
- **Use analogies**: "Like having a co-pilot for blockchain"
- **End strong**: The last 30 seconds should be memorable
